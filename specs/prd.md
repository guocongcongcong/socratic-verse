# Socratic Verse — 产品需求文档（PRD）

> 版本 v0.1 | 2026-06-16 | 锅釨

---

## 1. 产品愿景

**让世界上任何人都能以最低的意志力消耗，获得最高强度的提取式学习体验。**

2400年来，学习的终极矛盾一直在：最有效的方法需要另一个人，但最持久的学习必须是独处。AI 是这道题的第一次完整解答。Socratic Verse 要把这个解答做成一个开放的、可演化的系统。

---

## 2. 核心问题

| 问题 | 现状 | 我们的解法 |
|------|------|----------|
| 学习方法效率低 | 大多数人用重读（提取强度=0） | AI苏格拉底追问（提取强度=MAX） |
| 提取式学习意志力消耗高 | 合书复述/Feynman/交错练习需要自律 | AI 降低"开始"门槛（随机骰子/每日推送） |
| 苏格拉底教学法成本极高 | 只有顶尖私校能提供 | AI 以零边际成本复制 |
| 学习孤独 | 一个人学习没有人交流 | AI 角色扮演 + 对子教学 + 微信群聊 |
| 学习枯燥 | 纯文字/纯阅读 | 语音+捏脸+游戏化 |

---

## 3. 目标用户

| 用户画像 | 痛点 | Verse 解法 |
|---------|------|-----------|
| 自学程序员 | 教程看完了但写不出代码 | 苏格拉底追问直到能讲出"为什么" |
| 考研/考公党 | Anki 卡片只是死记硬背 | 推理型卡片 + 费曼检验 |
| 高中生 | 重读课本效率极低 | Cornell 笔记法起步 + 苏格拉底追问 |
| 终身学习者 | 买了书没看，看了记不住 | 合书复述 + 间隔重复 |
| 知识工作者 | 需要快速吃透新领域 | 自适应模式 20 分钟强力提取 |

---

## 4. 功能需求

### 4.1 文字模式（Phase 1-2）

| 功能 | 优先级 | 描述 |
|------|:---:|------|
| 教材导入 | P0 | 支持上传 markdown/pdf/txt，自动分段 |
| 苏格拉底教学引擎 | P0 | 8种教学方法，用户可选或自适应 |
| 学习进度追踪 | P0 | progress.md — 学到了哪里、哪些概念已掌握 |
| 教师角色切换 | P0 | 多位 AI 教师轮换上课 |
| 学习日记 | P1 | diary.md — 每节课后自动生成 |
| 微信群聊 | P1 | wechat_group.md — 多位老师课后互动 |
| Cornell 笔记生成 | P1 | AI 自动从教材生成关键词+总结 |
| Anki 卡片生成 | P1 | AI 从教材自动生成推理型卡片 |
| 知识图谱 | P2 | 概念间关系可视化 |

### 4.2 语音模式（Phase 3）

| 功能 | 优先级 | 描述 |
|------|:---:|------|
| TTS 语音输出 | P0 | edge-tts，多位角色的不同声线 |
| STT 语音输入 | P0 | Whisper，边走边答 |
| 打断机制 | P1 | 学习者在语音中可以说"等等"来暂停追问 |
| 离线语音 | P2 | 无网环境下本地 TTS+STT |

### 4.3 游戏模式（Phase 4）

| 功能 | 优先级 | 描述 |
|------|:---:|------|
| 捏脸系统 | P0 | ComfyUI/SD 生成自定义角色头像 |
| 角色动画 | P1 | Live2D 或简单表情系统 |
| 知识游戏生成 | P1 | AI 根据教材内容生成 HTML5 小游戏 |
| 游戏模板库 | P2 | 社区贡献的游戏模板（RPG对话树/问答战斗/解谜） |

### 4.4 平台与分发

| 功能 | 优先级 | 描述 |
|------|:---:|------|
| 终端客户端 | P0 | Claude Code / Hermes Agent 驱动 |
| Web 客户端 | P1 | React + Vite，浏览器直接运行 |
| 移动端 | P2 | 微信小程序 / PWA |
| Prompt 市场 | P2 | 社区分享教师角色、教材、方法版本 |
| Skill 包 | P1 | Hermes / Claude Code 可安装的 skill |

---

## 5. 非功能需求

| 需求 | 标准 |
|------|------|
| 零数据库 | 所有状态用 markdown 文件持久化 |
| 离线可用 | 文字模式完全离线（本地 LLM） |
| 多引擎 | 支持 Claude / GPT / DeepSeek / 本地 Ollama |
| 隐私 | 教材不上传云端（可选本地模式） |
| 开放 | 所有 Prompt 和角色文件开源，社区可贡献 |

---

## 6. 竞品分析

| 竞品 | 模式 | 我们的差异 |
|------|------|----------|
| Socratopia | 苏格拉底+二次元 | 我们加入8种方法选择 + 语音 + 游戏化 |
| Duolingo | 游戏化+间隔重复 | 我们做深度推理，不做浅层记忆 |
| Anki | 间隔重复 | 我们自动生成推理型卡片，且加入追问 |
| Khan Academy | 视频+练习 | 我们全程主动提取，不做被动观看 |
| ChatGPT | 回答模式 | 我们反转：AI追问你，不是你问AI |

---

## 7. 成功指标

| 指标 | 基线 | 目标 |
|------|:---:|:---:|
| 用户首次学习时长 | - | >20分钟（证明意志力门槛低） |
| 知识保留率（1周后） | 重读~20% | >70% |
| 用户回访率（日活） | - | >40% |
| NPS | - | >50 |

---

## 8. 风险

| 风险 | 影响 | 缓解 |
|------|:---:|------|
| LLM 追问质量不稳定 | 高 | 严格的 Prompt 工程设计 + 多模型 A/B 测试 |
| 语音延迟影响体验 | 中 | 流式 TTS，预加载常用追问 |
| 捏脸/游戏生成质量差 | 中 | Phase 4 之前不宣传，先做 POC 验证 |
| 用户不需要"学习方法选择" | 低 | 默认自适应模式，用户感知不到切换 |
