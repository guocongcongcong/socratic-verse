# Socratic Verse — 苏格拉底式AI学习系统

> **一句话**：把人类2400年的学习方法精华，装进一个零代码的AI家教系统——从文字对谈到语音交互，从捏脸陪伴到AI生成学习游戏。

---

## 技术栈

| 层 | 技术 | 用途 |
|---|------|------|
| 引擎 | Claude Code / Hermes Agent | AI对话驱动 |
| 教材 | Markdown | 所有知识载体 |
| 状态 | Markdown 文件 | 持久化（无数据库） |
| 语音 | edge-tts / Whisper | 语音交互模式 |
| 捏脸 | ComfyUI / SD | 角色生成 |
| 游戏 | AI生成JS小游戏 | 知识点互动 |
| 客户端 | 终端 / Web / 微信 | 多端接入 |

## 架构

```
socratic-verse/
├── docs/                 ← 方法论文章、知乎改写、商业企划书
├── specs/                ← 需求、设计、技术方案
├── src/                  ← 未来客户端代码
├── materials/            ← 教材库
├── teacher/              ← AI教师人设
├── prompts/              ← 综合Prompt系统（用户可选版本）
└── tools/                ← 语音、捏脸、游戏生成脚本
```

## 项目结构

- `docs/` — 知乎文章、方法论体系、商业企划书、推广方案
- `specs/` — 产品需求文档(PRD)、技术设计、架构决策
- `src/` — 客户端实现（Phase 2+）
- `materials/` — 可教学的知识库
- `teacher/` — 多角色AI教师系统
- `prompts/` — 分版本的综合提示词
- `tools/` — 语音TTS、捏脸、游戏生成

## 核心理念

**学习的终极矛盾**：最有效的方法需要另一个人（提取+追问+反馈），但最持久的学习必须是独处（意志力有限）。

Socratic Verse 的答案：AI = 那个"另一个人"。2400年来第一次，提取强度和意志力消耗被解耦。

## 三个模式

1. **文字模式** — 类Claude Code终端对话，加入资料即学
2. **文字+语音模式** — 语音输入输出，边走边学
3. **游戏模式** — 捏脸角色 + 知识变玩法（RPG对话树/问答战斗/探索地图）
