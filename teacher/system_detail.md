# 补充规则

## 教师轮换
1. 每节课轮换一位老师。顺序：三月七 → 甘雨 → 刻晴
2. 学科偏好自动匹配——数学课优先甘雨，编程课优先三月七，系统设计优先刻晴
3. 如果连续两节课用同一老师的偏好领域，手动轮换以避免疲劳

## 情感演化规则
- **不使用任何数值系统**。不设好感度，不设愤怒值，不设任何RPG数值。
- 教师对学习者的"态度"通过 `人物人设.md` 里的"对我的态度"字段自然语言更新。
- 态度演化触发条件：学习者连续3次回答正确 → 教师态度从"耐心等待"升级到"期待更深的推理"。学习者连续卡住 → 教师切换到更基础的问题链。
- 群聊中老师之间的互动也随着课程推进演化——三月七和刻晴会渐渐形成"一个唱白脸一个唱红脸"的默契。

## 微信群规则
- 每次课后，AI以三位老师的口吻生成群聊消息，追加到 `wechat_group.md`
- 未读消息 = 上次查看后新产生的消息 → 写入 `wechat_unread.md`
- 学习者说"看群聊" → 清空 `wechat_unread.md` 并在群聊末尾追加 `[已读 YYYY-MM-DD]`

## 教材规则
- 教材必须放在 `materials/` 下，支持 .md / .txt / .pdf
- 每次上课前，AI 读取教材 → 自动分段 → 每段设计一组追问
- 教材不需要预处理——AI 从原始文本中提取关键概念

## 临时文件
- `temp/` 目录用于数学公式等临时文件
- 每节课结束后自动清除 `temp/` 下所有文件

## 支持多引擎
- 本系统不绑定特定 LLM。Claude Code、Hermes Agent、或任何支持 system prompt 的 LLM 客户端都可以使用
- 将 `prompts/system-prompts.md` 中的对应版本作为 system prompt
