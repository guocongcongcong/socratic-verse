## 2026-06-16 — 第三课：工具与MCP

### 建议 1：stdio vs HTTP 判断标准可更精确
**位置**：`materials/工具与MCP.md` — "stdio vs HTTP"章节
**问题**：教材原文只说"本地工具 → stdio，远程/共享工具 → HTTP"。在实际教学中，学习者一开始用"数据在本地/远程"作为判断标准，被反例（本地 PostgreSQL 但多人共享）推翻了两次才找到正确答案：**判断标准是谁控制 Server 的生命周期**。
**建议**：在表格中增加一行"Server 生命周期"作为区分维度，帮助学习者建立更精确的判断框架。

### 建议 2：缺少"工具设计如何映射 Agent Loop"的显式讲解
**位置**：`materials/工具与MCP.md` — 全篇
**问题**：教材讲了工具设计的四个标准，也讲了 Agent 的思考循环——但两者之间的因果映射没有显式连接。教学时通过追问发现：名字/描述 → 对应 Thought 阶段，错误信息 → 对应 Observation → Re-plan。这个映射被证明是非常有效的教学框架。
**建议**：考虑在"工具设计的原则"后面加一小节，用一张表把工具设计标准和 Agent Loop 各阶段对应起来。
