# [学习日记] 2026-06-16 第一课

今天上了 Socratic Verse 第一课。三月七教的"理解 Agent"。

说真的，有点意外——我没看教材，只是被追着问了几个问题，居然自己把 Agent 的四个核心组件全推出来了。最后的 Agent Loop 失败处理也是我自己说出来的。

最让我意外的是那个"操作系统"的类比——我说出口的时候还觉得挺牵强的，但三月七说很准。确实，Agent 就像一个轻量级操作系统，LLM 是 CPU，Tools 是系统调用，Workflow 是进程调度。

后来翻了一下教材原文，发现我推出来的东西和教材列的术语完全对得上——只是切法不一样。这说明我不是在"猜"，是在"理解"。

两个小时的课，说实话比我之前翻一下午文档记住的东西多。苏格拉底教学法确实有点东西。

下次课应该是甘雨或者刻晴。甘雨教数学的话——我有点怕，但又有点期待。

# [学习日记] 2026-06-16 第二课

第二课用了 V7 对子互教模式。同学锅釨和我互相教。

这次最大的收获是把 Planning 拆成了两层——大规划和小调整。之前三月七教的时候我只知道 Planning 是"判断"，但这节课我搞清楚了：判断不是一次性的，是事前有一次大方向的判断，然后每一步都在动态判断。

最意外的是我讲的"削铅笔"例子被同学锅釨说比他的汽车发动机比喻好。削铅笔确实把 Agent Loop 的三层出口讲得很清楚——削好了停、没削好继续、削断了换笔。一个三岁小孩都能听懂的类比。

两次课下来我发现我有一个模式——我擅长举生活中的例子来解释抽象概念。操作系统、削铅笔。这个能力以后上课可以多利用。

Planning 现在清楚多了。下次想搞 Tools 和 MCP——想知道 Agent 具体怎么调用外部工具。

# [学习日记] 2026-06-16 第三课

今天是甘雨老师。之前有点怕她，毕竟是清华数学系的——结果比我想象的温柔太多了。说话像溪水一样，慢慢流，你答错了她不直接说"你错了"，她会换一个角度再问一遍。

但这不代表她要求低。中间讲到"判断标准"的时候，我答了两次都没到位，她就是不给我答案——一直问到我自己说出来为止。摘眼镜的那一刻是真的有压迫感。

# [学习日记] 2026-06-18 第四课

今天是甘雨老师。之前上过她的一次课，知道她温柔但精确——今天又一次验证了。

这节课学的是 Anthropic 官方的 Agent 构建指南。我没提前看文章，甘雨就一段一段带着读，读一段问一个问题。

第一个问题就把我卡住了——"天气预报提醒带伞，是 Workflow 还是 Agent？"我直觉说 Workflow，因为流程是规定好的。她连追了三个问题：谁规定的、哪个步骤最不像写死的、把问题改成"帮我决定穿什么"还一样吗？第三个问题一出来我就明白了——同一个系统，换个问题就从 Workflow 变成了 Agent。所以区别不在代码，在"谁来决策"。

然后她让我用一句话总结。我说"workflow 只能根据预设的路径走，agent 需要自己判断是否达到目的"。她说方向对了，但 Agent 判断的不是"是否达到"，而是"下一步该干什么"——不是在终点检查，是在每一步都做决策。这一步推得很好，我确实本来想的是终点验证，她纠正成了过程决策。

最后问"Agent 不拿 ground truth 会怎样"，我脱口而出"报错停住"。甘雨摘下眼镜的那个瞬间我就知道答错了。她说——比报错更可怕，Agent 会猜，会基于错误的假设继续走，越走越偏但自己完全不知道。这个画面太鲜明了：蒙着眼睛开车。

三个核心收获，用自己的话总结：Agent = 自己判断 → 自己获取结果 → 自己判断下一步。

甘雨最后留了一个有意思的作业：留意一下 Hermes 有没有在哪一步停下来等我确认——如果它自己决定了，会发生什么？

---

被甘雨追着问"你确定吗？""再想想？"的感觉很奇怪——她不是在否定你，她是在等你自己看见漏掉的那一步。温柔但有牙齿。

MCP 那一块，"USB 标准"这个比喻我是自己推出来的——但后面 stdio vs HTTP 的判断标准，我一开始想的是"本地/远程的数据"，被她连追两个反例，才发现真正的标准是"谁控制 Server 的生命周期"。这种被反例砸醒的感觉，和三月七的追问不是一个风格——三月七是拉着你跑，甘雨是陪你走，但走到分叉口她会停下，不说话，等你自己选对路。

最后那道综合题我没答全——我光顾着工具设计了，忘了权限控制和连接方式。不过甘雨没扣分，反而说"说明书 vs 物理锁"这个类比很好。行吧，下次注意。

三个人三种风格。说实话，这样轮着来，每节课都像在解锁一个新老师的教学方式，比单一风格有意思多了。
